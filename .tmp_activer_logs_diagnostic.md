## Comment activer les logs

### Option 1 : Variable d'environnement (recommandé)
Avant de lancer Streamlit :
```powershell
$env:LOG_LEVEL="INFO"
streamlit run ui/main.py
```

### Option 2 : Configuration logging dans le code
Si vous avez un fichier de config logging, assurez-vous que le niveau est INFO ou DEBUG.

### Option 3 : Logging direct dans console
Les logs apparaîtront dans :
- **Terminal Streamlit** (stdout/stderr)
- **Fichier de log** (si configuré dans `utils/logger.py`)

## Diagnostics possibles

### Scénario 1 : `auto_market_pick = ❌ DÉSACTIVÉ`
**Cause** : Le mode de sélection automatique n'est pas activé dans la sidebar.
**Solution** : Activer "Auto marché" dans les paramètres du Builder.

### Scénario 2 : `Marchés récents: ❌ VIDE` à chaque session
**Cause** : L'historique ne persiste pas (session Streamlit rafraîchie).
**Solution** : Implémenter la sauvegarde sur disque de `builder_autonomous_history`.

### Scénario 3 : Symbole toujours identique malgré `Recent markets (6): [...]`
**Cause** : Le LLM ignore les instructions de diversité OU l'override anti-répétition ne fonctionne pas.
**Solution** : Vérifier le log `LLM raw choice` pour voir si c'est le LLM qui choisit 0GUSDC ou l'override.

### Scénario 4 : `Shuffled symbols: 0G, ...` (0G toujours en premier après shuffle)
**Cause** : Possible biais de `random.shuffle()` (très improbable) OU univers très limité.
**Solution** : Vérifier `Univers disponible: N symboles` — si N < 5, l'univers est trop petit.

### Scénario 5 : `Valid symbol: NO` ou `Valid TF: NO`
**Cause** : Le LLM hallucine des symboles/timeframes inexistants.
**Solution** : Vérifier le fallback appliqué — le log `Marché sélectionné` montrera `Source: fallback_out_of_universe`.

Une fois le problème identifié, supprimez ou commentez les blocs :

# ── DIAG: ... ──
