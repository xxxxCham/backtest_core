# 🧭 Comportement du toggle "LLM choisit token/TF"

## Vue d'ensemble

Le toggle contrôle si le LLM choisit automatiquement le token et timeframe **pour chaque session Builder**, ou si vous utilisez les valeurs fixes de la sidebar.

---

## Toggle ✅ ACTIVÉ (par défaut en mode autonome)

### Processus
1. **Analyse de l'objectif** → Détecte le type de stratégie (scalping, breakout, momentum, trend, mean_reversion)
2. **Tri intelligent des tokens** → Classe les tokens par pertinence selon le type détecté
3. **Appel au LLM** → Demande au LLM de choisir le meilleur token/TF parmi la liste triée
4. **Chargement des données** → Charge automatiquement les données du marché sélectionné
5. **Affichage** → Montre le choix avec confidence et raison

### Exemple concret

**Objectif** : "Breakout Donchian sur crypto volatile 15m"

**Ce qui se passe** :
1. Détection : `breakout` → Tokens triés : APT, AVAX, SOL, BTC, ETH (haute volatilité en premier)
2. LLM reçoit le prompt :
   ```
   📊 RECOMMANDATION STRATÉGIE: Breakout détecté
     → TFs optimaux: 5m, 15m, 30m, 1h
     → Tokens prioritaires: APTUSDC, AVAXUSDC, SOLUSDC, BTCUSDC, ETHUSDC
     → IMPORTANT: Privilégier les tokens en tête de liste

   Contraintes:
   - symbol MUST be one of: APTUSDC, AVAXUSDC, SOLUSDC, BTCUSDC, ETHUSDC, ...
   - timeframe MUST be one of: 15m, 30m, 1h, 5m, ...
   ```
3. LLM choisit : `APTUSDC 15m` (conf=0.85)
4. Données chargées : `APTUSDC_15m.parquet`

**Affichage dans l'UI** :
```
🧭 Session #1: APTUSDC 15m (source=llm, data=loaded, conf=0.85)
```

### Avantages
✅ **Pertinence maximale** : Token/TF adapté au type de stratégie
✅ **Diversification automatique** : Évite de répéter les mêmes marchés (système de diversité)
✅ **Exploration large** : Teste différents tokens/TF à chaque session
✅ **Pas de biais humain** : Le LLM choisit objectivement selon les critères

### Logs dans la console
```
Market selection: strategy_type=breakout, ranked_tokens=APTUSDC, AVAXUSDC, SOLUSDC, BTCUSDC, ETHUSDC
Market selection: fallback=APTUSDC 15m (source=strategy_optimized)
🔍 [DIAG] Session #1 → Marché sélectionné: APTUSDC 15m | Source: llm | Confidence: 0.85 | Data: loaded
```

## Toggle ❌ DÉSACTIVÉ

### Processus
1. **Utilise les valeurs de la sidebar** : Token et TF sélectionnés manuellement
2. **Pas d'appel au LLM** pour la sélection de marché
3. **Même marché pour toutes les sessions** (sauf si vous changez manuellement)

### Exemple concret

**Sidebar** : `BTCUSDC` + `1h`
**Objectif** : "Breakout Donchian sur crypto volatile 15m"

**Ce qui se passe** :
1. ❌ Pas de détection du type de stratégie
2. ❌ Pas d'analyse de pertinence
3. ✅ Utilise directement `BTCUSDC 1h` (même si 15m est demandé dans l'objectif)
4. ✅ Données chargées : `BTCUSDC_1h.parquet` (déjà en cache)

**Affichage dans l'UI** :
Rien de spécial, juste le builder qui démarre avec BTCUSDC 1h

### Avantages
✅ **Contrôle total** : Vous décidez exactement quel marché tester
✅ **Reproductibilité** : Toutes les sessions utilisent le même marché
✅ **Rapide** : Pas d'appel LLM supplémentaire, pas de chargement de données

### Logs dans la console
```
🔍 [DIAG] Session #1 → Marché PAR DÉFAUT (auto_market_pick=OFF): BTCUSDC 1h
```

### ⚠️ Limitations
- **Pas de diversification** : Toujours le même token/TF
- **Pas adapté au type de stratégie** : Peut tester un breakout sur un token à faible volatilité
- **Incohérence avec l'objectif** : Si l'objectif dit "15m" mais sidebar = "1h", c'est 1h qui gagne

## Comparaison visuelle

| Aspect | Toggle ✅ ACTIVÉ | Toggle ❌ DÉSACTIVÉ |
|--------|-----------------|---------------------|
| **Token/TF source** | Choisi par LLM selon stratégie | Sidebar (valeurs fixes) |
| **Pertinence stratégie** | ✅ Optimale (tri intelligent) | ⚠️ Aléatoire (dépend de vous) |
| **Diversification** | ✅ Automatique (évite répétitions) | ❌ Aucune (toujours pareil) |
| **Chargement données** | Auto (peut être lent si nouveau token) | Rapide (déjà en cache) |
| **Affichage UI** | `🧭 Session #1: APTUSDC 15m (source=llm, ...)` | Rien (utilise sidebar) |
| **Logs console** | Détaillés (stratégie, ranking, confidence) | Minimal |
| **Appels LLM** | +1 par session (sélection marché) | 0 |
| **Use case** | Mode autonome 24/24, exploration | Tests reproductibles, debugging |

---

## Quand utiliser chaque mode ?

### Toggle ✅ ACTIVÉ → Recommandé pour :
- **Mode autonome 24/24** : Exploration automatique de différents marchés
- **Tests multi-stratégies** : Chaque stratégie obtient le token/TF optimal
- **Diversification** : Éviter de tester 100x le même marché
- **Découverte** : Laisser le système trouver les meilleurs marchés

### Toggle ❌ DÉSACTIVÉ → Recommandé pour :
- **Tests ciblés** : "Je veux tester uniquement BTCUSDC 1h"
- **Debugging** : Isoler un marché spécifique pour reproduire un bug
- **Comparaisons** : Tester 10 stratégies sur le même marché
- **Performance** : Éviter les appels LLM supplémentaires

## Impact du fix récent

### Avant le fix (même avec toggle activé)
```
Session #1 → 0GUSDC 1h (toujours pareil)
Session #2 → 0GUSDC 1h (toujours pareil)
Session #3 → 0GUSDC 1h (toujours pareil)
...avec override après coup...
```

### Après le fix (toggle activé)
```
Session #1 → APTUSDC 30m (breakout optimal dès le départ)
Session #2 → BTCUSDC 4h (trend optimal dès le départ)
Session #3 → SOLUSDC 5m (scalping optimal dès le départ)
```

**Le fix a rendu le toggle vraiment fonctionnel** : maintenant quand il est activé, le LLM fait des choix intelligents dès le départ, au lieu de toujours retourner 0GUSDC 1h.

## Configuration recommandée

**Mode autonome 24/24** :
- ✅ Toggle activé
- ✅ Multi-symbols dans sidebar (diversification maximale)
- ✅ Multi-timeframes dans sidebar (flexibilité)
- ✅ Objectifs LLM ou paramétriques (diversité de stratégies)

**Tests manuels** :
- ❌ Toggle désactivé
- ✅ 1 symbol + 1 timeframe dans sidebar
- ✅ Objectif manuel spécifique

---

**Résumé** : Le toggle contrôle si le **LLM choisit intelligemment le marché selon la stratégie** (activé) ou si vous **imposez un marché fixe** (désactivé). Avec le fix récent, le mode activé fonctionne enfin correctement !
